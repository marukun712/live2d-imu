# @kokoro/rig
[![JSR](https://jsr.io/badges/@kokoro/rig)](https://jsr.io/@kokoro/rig)

PSD レイヤーをメッシュ変形でアニメーションさせる、人型キャラクター向けのリギングライブラリです。
ボーン構造を使わず、コードと頂点ウェイトの定義だけでキャラクターを動かすことを目的に設計されています。

## コンセプト

### キャラクターを「数値の配列」として扱う

3D・2D 問わず、キャラクターのアニメーションには一般的にボーン構造が使われます。
ボーンはアニメータにとってなじみ深い概念ですが、エンジニアにとっては必ずしもそうではありません。

`@kokoro/rig` は別のアプローチをとります。
キャラクターを構成する全スプライトのメッシュ頂点を、一本の `Float32Array` として扱います。
「キャラクターを動かす」とは、この配列の値を毎フレーム書き換えることです。

### ウェイト関数で動かす

各頂点には UV 座標 (u, v) が割り当てられています。u=0 が左端・u=1 が右端、v=0 が上端・v=1 が下端です。
ポーズは `(u, v, t) => { tx, ty, rot, w }` という形の関数で表現します。

- `tx`, `ty` : その頂点をどれだけ平行移動するか
- `rot` : どれだけ回転させるか
- `w` : このポーズのウェイト (0~1)

ウェイト `w` に UV 座標を使うことで、「上半身ほど大きく動く」「毛先ほどよく揺れる」といった
位置に連動した自然な変形を1つの関数で表現できます。

```ts
// 「左を向く」ポーズの例。頭に近いほど大きく動く。
left: (u, v) => ({
  tx: -100,
  rot: -0.05,
  w: curve.body(1 - v),
})
```

`curve` は UV 値をウェイトに変換するイージング関数の辞書です。
ウェイトというアイデアから自然に生まれた概念で、「どの頂点がどれだけ動くか」を直感的に調整できます。

### ポーズの補間と合成

複数のポーズを同時に `setPose` に渡すと、ウェイトに従って加算合成されます。
`lerpBlend` で2つのポーズを線形補間した新しいポーズを作ることもできます。

```ts
rig.setPose([
  rig.lerpBlend("left", "right", mouseX),
  rig.lerpBlend("up", "down", mouseY),
  rig.lerpBlend("normal", "breathing", breathCycle),
]);
```

キャラクターの動きは、数値のブレンドとして表現されます。

### 部位ごとのリグと親子関係

体幹・髪・腕などに個別のリグを作り、`parent` で体幹リグを親として指定できます。
子リグは親の変形を継承しつつ、独自のポーズ (髪の揺れなど) を上乗せします。

## 開発

```sh
bun install
bun dev  # デモアプリを起動
```

デモアプリの実行には `public/models/character.psd` にキャラクター PSD を配置する必要があります。

```sh
bun psd ./public/models/character.psd  # PSD のレイヤー構造を確認する
```
